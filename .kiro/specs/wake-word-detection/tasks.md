## Manual Prerequisites（手作業）

- [ ] 0.1 Picovoice Consoleでアカウントを作成し、Access Keyを取得する
  - https://console.picovoice.ai/ でアカウント作成（個人利用無料）
  - Access Keyをコピーし控えておく（実装後に設定画面から入力する）

- [ ] 0.2 カスタムウェイクワード「TONaRi」のモデルファイルを作成する
  - Picovoice Consoleの Porcupine > Custom Keywords で「TONaRi」を作成
  - ターゲットプラットフォーム: **Web (WASM)** を選択
  - 言語: **Japanese** を選択
  - 生成された `.ppn` ファイルをダウンロード
  - `public/models/porcupine/` に配置する

- [ ] 0.3 Porcupine言語モデルファイル(.pv)を取得する
  - Picovoice ConsoleまたはGitHubリポジトリから日本語用の `.pv` モデルファイルをダウンロード
  - `public/models/porcupine/` に配置する

- [ ] 0.4 ウェイクワード検知通知音を用意する
  - 短い効果音(.mp3)を用意する（フリー素材またはオリジナル）
  - `public/sounds/` に配置する

## Tasks

- [ ] 1. (P) Porcupine React SDKのインストールとディレクトリ構成の準備
  - `@picovoice/porcupine-react`パッケージをインストールする
  - `public/models/porcupine/` ディレクトリを作成し、手作業（0.2, 0.3）で配置されたモデルファイルの読み込みを確認する
  - `public/sounds/` に手作業（0.4）で配置された効果音の読み込みを確認する
  - _Requirements: 1.4_

- [ ] 2. 状態管理の基盤構築
- [ ] 2.1 (P) 設定ストアにウェイクワード関連設定を追加する
  - ウェイクワード検知のオン/オフ設定（デフォルト: オフ）をlocalStorageに永続化する
  - Picovoice Access Keyの保存欄を追加し、localStorageに永続化する
  - _Requirements: 6.2_

- [ ] 2.2 (P) 音声入力の状態管理ストアを新規作成する
  - 音声入力フェーズ（idle/listening/processing/awaitingFollowUp/disabled）を管理する
  - 中間テキスト、確定テキスト、入力フォーム書き込み用テキストの各バッファを保持する
  - バッファへの追記（サイレンスタイマーによるSTT再開時）とリセット操作を提供する
  - _Requirements: 2.2, 3.2, 4.3, 4.4_

- [ ] 3. (P) Web Speech APIラッパーサービスの作成
  - ブラウザの音声認識APIを抽象化し、開始・停止・中断のインターフェースを提供する
  - 日本語音声認識（ja-JP）に設定し、中間結果と確定結果をコールバックで通知する
  - 音声認識の終了イベント（speechend）を呼び出し元に通知し、サイレンスタイマー制御を委譲する
  - ブラウザの音声認識API非対応を判定するプロパティを提供する
  - 将来のSTTエンジン差し替えに備えたインターフェースを維持する
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [ ] 4. ウェイクワード検知フックの実装
- [ ] 4.1 ウェイクワード検知とSTT開始の基本フローを実装する
  - Porcupineのウェイクワード検知イベントを受信し、検知時に効果音を再生する
  - 検知時にアバターが睡眠モーション中であれば起こす
  - Porcupineのマイクリスニングを停止し、音声認識サービスを開始する（マイク逐次切り替え）
  - 設定ストアのウェイクワード有効/無効に応じてPorcupineの初期化・開始・停止を制御する
  - Access Key未設定やマイク許可拒否時のエラーハンドリングを行い、適切に通知する
  - Web Speech API非対応ブラウザでの部分動作（ウェイクワード検知のみ有効）を実装する
  - ページリロード時に保存された設定に基づいてウェイクワード検知を自動開始する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.5, 4.1, 4.2, 5.1, 5.2, 5.3, 5.4, 6.3_

- [ ] 4.2 サイレンスタイマー付きSTT再開ループを実装する
  - 音声認識のspeechend発火時に3秒のサイレンスタイマーを開始する
  - タイマー期限内にユーザーが再度発話した場合、STTを再開しバッファに追記して会話を継続する
  - 3秒間完全に沈黙した場合、バッファを確定する
  - バッファが空の状態で沈黙タイムアウトした場合、送信せずにウェイクワード待機に復帰する
  - _Requirements: 2.3, 3.4, 5.4_

- [ ] 4.3 テキスト送信ロジックを実装する
  - バッファ確定後、Tonariのタブにフォーカスがない場合は既存のチャット送信関数で自動送信する
  - Tonariのタブにフォーカスがある場合はバッファ内容を入力フォームに書き込み、ユーザーが編集・送信できる状態にする
  - 送信後にバッファをクリアする
  - _Requirements: 3.1, 3.2, 3.3_

- [ ] 4.4 会話モード（エージェント応答後の自動STT再開）を実装する
  - テキスト送信後、エージェントの応答処理とTTS再生の両方が完了するまで待機する
  - TTS完了後に自動で音声認識を再開し、10秒のフォローアップタイマーを開始する
  - ユーザーが話し始めた場合、タイマーをキャンセルしリスニング状態に戻す（会話続行）
  - 10秒間沈黙した場合、音声認識を停止しPorcupineのウェイクワード待機に復帰する

- [ ] 5. UIコンポーネントの実装
- [ ] 5.1 (P) 音声入力インジケーターを実装する
  - リスニング中のマイクアイコンとパルスアニメーションを表示する
  - 音声認識中のテキストをリアルタイムにプレビュー表示する
  - フォローアップ待機中（awaitingFollowUp）の状態表示を行う
  - フェーズに応じた表示切り替え（listening/processing/awaitingFollowUp/非表示）を行う
  - _Requirements: 4.3, 4.4_

- [ ] 5.2 (P) ウェイクワード設定パネルを実装する
  - ウェイクワード検知のオン/オフトグルを設定画面に追加する
  - Picovoice Access Keyの入力欄を提供する
  - Access Keyが未設定の状態でトグルONを試みた場合に入力を促すメッセージを表示する
  - _Requirements: 6.1, 6.4, 6.5_

- [ ] 6. ページへの統合とエンドツーエンド動作確認
  - useWakeWordDetectionフックとVoiceInputIndicatorをメインページにマウントする
  - WakeWordSettingsを既存の設定画面に組み込む
  - ウェイクワード検知 → 音声入力 → テキスト送信 → エージェント応答 → 会話モードのフルフローを検証する
  - _Requirements: 1.1, 2.1, 3.1, 3.3, 4.1, 4.3_
